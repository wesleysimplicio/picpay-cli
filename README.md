   157|## Install paths
   158|
   159|### A. `npx` — recommended, cross-platform, zero clone
   160|
   161|```bash
   162|# inside your project folder (works on macOS, Linux, Windows)
   163|npx @wesleysimplicio/llm-project-mapper
   164|```
   165|
   166|Runs interactively. Asks **only**:
   167|
   168|1. **Which CLI/LLM to hand off to after the automatic mapping pass** (auto-detects which ones are installed and marks them `[installed]`).
   169|2. **Append recommended ignores to `.gitignore`?** (yes/no — never overwrites your existing `.gitignore`).
   170|
   171|Everything else — `PRODUCT_NAME`, stack, dependencies — auto-detected from `package.json` / `pyproject.toml` / `go.mod` / `*.csproj` / `Cargo.toml` / `pubspec.yaml` / `composer.json` / `Gemfile` / `mix.exs` / `pom.xml` / `build.gradle*`.
   172|
   173|#### Non-interactive (CI / scripts)
   174|
   175|```bash
   176|npx @wesleysimplicio/llm-project-mapper --yes --cli skip --append-gitignore no
   177|```
   178|
   179|#### Update an existing starter overlay
   180|
   181|```bash
   182|npx @wesleysimplicio/llm-project-mapper@latest --update
   183|```
   184|
   185|This is equivalent to `--yes --force --append-gitignore yes --cli skip`: it refreshes starter-managed files, updates the starter `.gitignore` block, preserves existing instruction files, and does not launch an agent.
   186|
   187|#### Preview without writing
   188|
   189|```bash
   190|npx @wesleysimplicio/llm-project-mapper --dry-run --yes
   191|```
   192|
   193|#### Full flag list
   194|
   195|| Flag | Purpose |
   196||---|---|
   197|| `-y, --yes` | Non-interactive (defaults: no `.gitignore` append, skip CLI handoff) |
   198|| `-f, --force` | Overwrite starter template files. **Never** touches user instruction files (`AGENTS.md`, `CLAUDE.md`, `INIT.md`, `.github/copilot-instructions.md`, `.gitignore`) |
   199|| `--update` | Safe update mode for an existing overlay: force starter files, update `.gitignore`, skip handoff |
   200|| `--dry-run` | Print actions without writing |
   201|| `--cli <key>` | Pick CLI for `INIT.md` handoff: `claude`, `codex`, `copilot`, `cursor`, `deepseek`, `kimi`, `minimax`, `glm`, `hermes`, `openclaw`, `aider`, `other`, `skip` |
   202|| `--append-gitignore <yes\|no>` | Append recommended ignores to `.gitignore` |
   203|| `--skip-meta` | Do not write `.starter-meta.json` |
   204|| `--silent` | Minimal output |
   205|| `-v, --version` | Print version |
   206|| `-h, --help` | Show help |

   278|## What `INIT.md` does — the safety contract
   279|
   280|When the chosen CLI runs `INIT.md`, it reads `.starter-meta.json` and follows three hard rules:
   281|
   282|1. **`read_only_globs` are intouchable.** Any file matching these globs (`**/*.razor`, `**/*.cs`, `**/*.csproj`, `**/*.sln`, `package.json`, `pnpm-lock.yaml`, `yarn.lock`, `package-lock.json`, `**/*.py`, `**/*.go`, `**/*.rs`, `**/*.java`, `**/*.kt`, `**/*.dart`, `**/*.php`, `**/*.rb`) is read-only. The agent reads it for context but never writes. If `git status` shows any after init — that is a bug.
   283|2. **`init_must_merge` preserves your essence.** If `AGENTS.md` / `CLAUDE.md` / `.github/copilot-instructions.md` already existed before bootstrap, the agent **reads them**, **preserves the content**, and **merges** the starter structure on top. Never a clean rewrite.
   284|3. **`init_must_ask` only asks 4 things.** `team`, `domain`, `vision_oneliner`, `primary_personas` — once, in a single message. Everything else (`product_name`, `stack`) is auto-detected.
   285|
   286|The agent then writes — and only writes — inside the whitelist:
   287|
   288|```
   289|.specs/**          .agents/**         .skills/**
   290|.claude/**         .codex/**
   291|.github/copilot-instructions.md
   292|.github/copilot/**
   293|.github/PULL_REQUEST_TEMPLATE.md
   294|.github/ISSUE_TEMPLATE/**
   295|.github/workflows/ci.yml
   296|.github/workflows/dod.yml
   297|AGENTS.md  CLAUDE.md  README.md  README.pt-BR.md
   298|playwright.config.ts (only if missing or our template)
   299|```
   300|
   301|Anything outside this whitelist **and** not from the starter template = untouched.

   305|## Troubleshooting
   306|
   307|### macOS / Linux
   308|
   309|| Symptom | Fix |
   310||---|---|
   311|| `./bootstrap.sh: Permission denied` | `chmod +x ./bootstrap.sh` |
   312|| `command not found: npx` | Install Node.js (see Prerequisites) |
   313|| `Claude Code not installed` after pick | Install Claude Code or pick `[12] Other` to copy the prompt to clipboard |
   314|| Old Bash on macOS (`bash --version` shows 3.2) | Works — script is Bash 3.2-compatible. If problems, `brew install bash` for Bash 5+ |
   315|
   316|### Windows
   317|
   318|| Symptom | Fix |
   319||---|---|
   320|| `bootstrap.ps1 cannot be loaded ... execution policy` | Run with `powershell -ExecutionPolicy Bypass -File .\bootstrap.ps1` (per-session bypass, no permanent change) |
   321|| Line endings broken when running `.sh` from Git Bash | `git config --global core.autocrlf input` then re-clone |
   322|| `npx` not found in cmd.exe | Open new terminal after Node install (refreshes PATH), or use full path `C:\Program Files\nodejs\npx.cmd` |
   323|| `pwsh` not found | You have PowerShell 5.1 (built-in) — use the `powershell -ExecutionPolicy Bypass ...` form. To install pwsh 7: `winget install Microsoft.PowerShell` |
   324|
   325|### Cross-platform
   326|
   327|| Symptom | Fix |
   328||---|---|
   329|| Bootstrap exits with `aborting: existing files would be overwritten` | Re-run with `--force` (only overwrites starter template files, never your instruction files) |
   330|| `git status` shows `package.json` / source files modified after init | Stop. That is a `read_only_globs` violation. Open an issue with the file path |
   331|| `.gitignore` got rewritten | The starter never overwrites it — only appends if you said `yes`. If yours was replaced, you ran `--force`; restore from git |
   332|| Want to re-run `INIT.md` later | `claude "$(cat INIT.md)"` (or equivalent for your CLI). The handoff is just a launcher |

   388|## Philosophy
   389|
   390|- **Specs as code.** What is not in `.specs/`, the agent does not see.
   391|- **Atomic tasks.** One task = one small reviewable PR.
   392|- **Automated DoD.** What does not pass the gate, does not merge.
   393|- **Reusable skills.** A capability that becomes a pattern becomes a `SKILL.md`.
   394|- **Tight loop.** Edit, test, fix, repeat. Never accumulate invisible debt.
   395|- **Never destroy.** User files are read-only; starter files merge instead of overwrite.
