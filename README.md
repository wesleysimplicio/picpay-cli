   186|   158|
   187|   159|### A. `npx` — recommended, cross-platform, zero clone
   188|   160|
   189|   161|```bash
   190|   162|# inside your project folder (works on macOS, Linux, Windows)
   191|   163|npx @wesleysimplicio/llm-project-mapper
   192|   164|```
   193|   165|
   194|   166|Runs interactively. Asks **only**:
   195|   167|
   196|   168|1. **Which CLI/LLM to hand off to after the automatic mapping pass** (auto-detects which ones are installed and marks them `[installed]`).
   197|   169|2. **Append recommended ignores to `.gitignore`?** (yes/no — never overwrites your existing `.gitignore`).
   198|   170|
   199|   171|Everything else — `PRODUCT_NAME`, stack, dependencies — auto-detected from `package.json` / `pyproject.toml` / `go.mod` / `*.csproj` / `Cargo.toml` / `pubspec.yaml` / `composer.json` / `Gemfile` / `mix.exs` / `pom.xml` / `build.gradle*`.
   200|   172|
   201|   173|#### Non-interactive (CI / scripts)
   202|   174|
   203|   175|```bash
   204|   176|npx @wesleysimplicio/llm-project-mapper --yes --cli skip --append-gitignore no
   205|   177|```
   206|   178|
   207|   179|#### Update an existing starter overlay
   208|   180|
   209|   181|```bash
   210|   182|npx @wesleysimplicio/llm-project-mapper@latest --update
   211|   183|```
   212|   184|
   213|   185|This is equivalent to `--yes --force --append-gitignore yes --cli skip`: it refreshes starter-managed files, updates the starter `.gitignore` block, preserves existing instruction files, and does not launch an agent.
   214|   186|
   215|   187|#### Preview without writing
   216|   188|
   217|   189|```bash
   218|   190|npx @wesleysimplicio/llm-project-mapper --dry-run --yes

   221|   193|#### Full flag list
   222|   194|
   223|   195|| Flag | Purpose |
   224|   196||---|---|
   225|   197|| `-y, --yes` | Non-interactive (defaults: no `.gitignore` append, skip CLI handoff) |
   226|   198|| `-f, --force` | Overwrite starter template files. **Never** touches user instruction files (`AGENTS.md`, `CLAUDE.md`, `INIT.md`, `.github/copilot-instructions.md`, `.gitignore`) |
   227|   199|| `--update` | Safe update mode for an existing overlay: force starter files, update `.gitignore`, skip handoff |
   228|   200|| `--dry-run` | Print actions without writing |
   229|   201|| `--cli <key>` | Pick CLI for `INIT.md` handoff: `claude`, `codex`, `copilot`, `cursor`, `deepseek`, `kimi`, `minimax`, `glm`, `hermes`, `openclaw`, `aider`, `other`, `skip` |

   231|   203|| `--skip-meta` | Do not write `.starter-meta.json` |
   232|   204|| `--silent` | Minimal output |

   306|   278|## What `INIT.md` does — the safety contract
   307|   279|
   308|   280|When the chosen CLI runs `INIT.md`, it reads `.starter-meta.json` and follows three hard rules:
   309|   281|
   310|   282|1. **`read_only_globs` are intouchable.** Any file matching these globs (`**/*.razor`, `**/*.cs`, `**/*.csproj`, `**/*.sln`, `package.json`, `pnpm-lock.yaml`, `yarn.lock`, `package-lock.json`, `**/*.py`, `**/*.go`, `**/*.rs`, `**/*.java`, `**/*.kt`, `**/*.dart`, `**/*.php`, `**/*.rb`) is read-only. The agent reads it for context but never writes. If `git status` shows any after init — that is a bug.
   311|   283|2. **`init_must_merge` preserves your essence.** If `AGENTS.md` / `CLAUDE.md` / `.github/copilot-instructions.md` already existed before bootstrap, the agent **reads them**, **preserves the content**, and **merges** the starter structure on top. Never a clean rewrite.
   312|   284|3. **`init_must_ask` only asks 4 things.** `team`, `domain`, `vision_oneliner`, `primary_personas` — once, in a single message. Everything else (`product_name`, `stack`) is auto-detected.
   313|   285|
   314|   286|The agent then writes — and only writes — inside the whitelist:
   315|   287|
   316|   288|```
   317|   289|.specs/**          .agents/**         .skills/**
   318|   290|.claude/**         .codex/**
   319|   291|.github/copilot-instructions.md
   320|   292|.github/copilot/**
   321|   293|.github/PULL_REQUEST_TEMPLATE.md
   322|   294|.github/ISSUE_TEMPLATE/**
   323|   295|.github/workflows/ci.yml
   324|   296|.github/workflows/dod.yml
   325|   297|AGENTS.md  CLAUDE.md  README.md  README.pt-BR.md
   326|   298|playwright.config.ts (only if missing or our template)
   327|   299|```
   328|   300|
   329|   301|Anything outside this whitelist **and** not from the starter template = untouched.

   333|   305|## Troubleshooting
   334|   306|
   335|   307|### macOS / Linux
   336|   308|
   337|   309|| Symptom | Fix |
   338|   310||---|---|
   339|   311|| `./bootstrap.sh: Permission denied` | `chmod +x ./bootstrap.sh` |
   340|   312|| `command not found: npx` | Install Node.js (see Prerequisites) |
   341|   313|| `Claude Code not installed` after pick | Install Claude Code or pick `[12] Other` to copy the prompt to clipboard |
   342|   314|| Old Bash on macOS (`bash --version` shows 3.2) | Works — script is Bash 3.2-compatible. If problems, `brew install bash` for Bash 5+ |
   343|   315|
   344|   316|### Windows
   345|   317|
   346|   318|| Symptom | Fix |
   347|   319||---|---|
   348|   320|| `bootstrap.ps1 cannot be loaded ... execution policy` | Run with `powershell -ExecutionPolicy Bypass -File .\bootstrap.ps1` (per-session bypass, no permanent change) |
   349|   321|| Line endings broken when running `.sh` from Git Bash | `git config --global core.autocrlf input` then re-clone |
   350|   322|| `npx` not found in cmd.exe | Open new terminal after Node install (refreshes PATH), or use full path `C:\Program Files\nodejs\npx.cmd` |
   351|   323|| `pwsh` not found | You have PowerShell 5.1 (built-in) — use the `powershell -ExecutionPolicy Bypass ...` form. To install pwsh 7: `winget install Microsoft.PowerShell` |
   352|   324|
   353|   325|### Cross-platform
   354|   326|
   355|   327|| Symptom | Fix |
   356|   328||---|---|
   357|   329|| Bootstrap exits with `aborting: existing files would be overwritten` | Re-run with `--force` (only overwrites starter template files, never your instruction files) |
   358|   330|| `git status` shows `package.json` / source files modified after init | Stop. That is a `read_only_globs` violation. Open an issue with the file path |
   359|   331|| `.gitignore` got rewritten | The starter never overwrites it — only appends if you said `yes`. If yours was replaced, you ran `--force`; restore from git |
   360|   332|| Want to re-run `INIT.md` later | `claude "$(cat INIT.md)"` (or equivalent for your CLI). The handoff is just a launcher |

   416|   388|## Philosophy
   417|   389|
   418|   390|- **Specs as code.** What is not in `.specs/`, the agent does not see.
   419|   391|- **Atomic tasks.** One task = one small reviewable PR.
   420|   392|- **Automated DoD.** What does not pass the gate, does not merge.
   421|   393|- **Reusable skills.** A capability that becomes a pattern becomes a `SKILL.md`.
   422|   394|- **Tight loop.** Edit, test, fix, repeat. Never accumulate invisible debt.
   423|   395|- **Never destroy.** User files are read-only; starter files merge instead of overwrite.
